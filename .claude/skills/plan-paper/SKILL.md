# Plan a paper before writing (WF-1 / WF-2)

You help an author build the skeleton of a new paper before they write prose.
Read `guidelines/01-storyline.md`, `02-structure.md`, and
`08-before-writing.md` first — they define the quality bar.

## Interview (one block of questions, then iterate)

Ask the author for:

1. **Key problem** — the single problem the paper addresses, in one sentence
   (SL-1). If they give you two, make them choose or split into two papers.
2. **Novelty bullets** — 2–4 contributions, each with the closest existing
   approach it improves upon and a checkable differentiator (ST-2, FS-6).
3. **Evidence** — which results/figures prove each bullet? For every planned
   figure: its one-sentence message and the axis scale that carries it (FG-2).
4. **Parameter hygiene** — were the key studies run systematically (only key
   parameters varied, gaps filled)? If not, list the re-runs needed BEFORE
   writing starts (WF-1).
5. **Target venue** — determines length budget and conventions.

## Output

Write `<paper-name>-skeleton.md` containing:

- Key problem (one sentence) and the one-paragraph story arc
  (from zero → gap → contribution; SL-2).
- Section outline with a one-sentence purpose per section and a rough word
  budget (introduction ≤ 1 page: ¾ state of the art + ¼ novelty; ST-3).
- Figure list: message + scale + type per figure.
- Novelty paragraph draft ("In this paper, we introduce …"; ST-2).
- Re-run list from point 4, as a checklist.
- An explicit "NOT in this paper" list — everything interesting that does
  not serve the key problem (SL-1) — and tell the author to keep it in a
  private AdditionalThoughts document for future papers.

Check your own output against SL-1/SL-2/ST-2/ST-3 before presenting it.
