# Rule proposals

Landing zone for new rules distilled from manuscript/revision feedback.

One file per proposal, named `YYYY-MM-DD-short-slug.md`, following the rule
format of [`00-meta.md`](../00-meta.md) plus a mandatory **Provenance** line
and the raw feedback excerpt it was distilled from (anonymized if needed).

Proposals are reviewed by a maintainer and then either

- **merged** into the matching category file (next free ID),
- **folded** into an existing rule (as an additional example or sharpened
  wording), or
- **rejected** with a short note (kept here for the record, prefixed
  `rejected-`).

Never add rules directly to the category files without passing through this
directory — the curation step is what keeps the catalogue trustworthy.
